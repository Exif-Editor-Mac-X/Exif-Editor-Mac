## 🔍 Overview

<a href="#exifEditor">Exif Editor</a> serves photographers who need professional metadata management, privacy-conscious users who want to control what personal information is embedded in their images before sharing, stock photographers who need standardized metadata for submissions, and anyone managing photo collections where metadata accuracy matters.

The <a href="#exifEditor">professional photographer metadata workflow</a> embeds copyright, author, contact information, and usage rights into every image before delivery — the industry standard for professional image distribution that ensures ownership information travels with the image through editorial, stock, and licensing workflows. Batch embedding applies this information to entire shoot deliveries without per-image manual entry. The <a href="#exifEditor">privacy protection use case</a> serves anyone sharing photos online who doesn't want to reveal their precise location, camera model, or other personal information embedded in image metadata. Images shared on social media, sent by email, or posted on websites carry EXIF metadata that GPS-enabled cameras embed automatically — Exif Editor removes this information before sharing.

The <a href="#exifEditor">photo collection organization use case</a> corrects metadata errors accumulated in photo libraries — wrong timestamps from camera clock errors, missing location data that should have been captured, inconsistent copyright fields across a library — bringing metadata accuracy to photo collections. <a href="#exifEditor">Stock photography submission preparation</a> uses IPTC metadata fields for keywords, captions, and rights management information that stock agencies require for image submission and cataloging.

<a href="#exifEditor">IPTC metadata editing</a> provides the editorial metadata fields used by news agencies, stock libraries, and publication workflows — caption, headline, keywords, credit line, source, copyright notice, and contact information fields standardized for professional image distribution. <a href="#exifEditor">XMP metadata support</a> reads and writes Adobe's XMP (Extensible Metadata Platform) metadata standard used by Lightroom, Photoshop, and other professional photography applications.

<a href="#exifEditor">Side-car file support</a> handles XMP sidecar files that store metadata separately from RAW image files — editing metadata through sidecar files for RAW formats that don't support embedded metadata modification without altering the original RAW data. <a href="#exifEditor">Mac App Store availability</a> provides standard installation through Apple's App Store with Apple's standard security review and update delivery.

## ❓ FAQ

<details>
<summary>How does Exif Editor compare to ExifTool?</summary>

ExifTool is a command-line utility with the broadest possible metadata support. Exif Editor provides a graphical Mac interface for the most common metadata operations — viewing, editing, and batch processing — making metadata work accessible without command-line knowledge.

</details>

<details>
<summary>Does Exif Editor preserve image quality when editing metadata?</summary>

Yes. Metadata editing modifies only the embedded metadata fields without altering image pixel data or recompressing the image.

</details>

<details>
<summary>Can I export metadata from photos using Exif Editor?</summary>

Yes. Metadata exports to tabular formats for documentation and archival.

</details>

<details>
<summary>Does Exif Editor run on Apple Silicon?</summary>

Yes. Current versions support Apple Silicon Macs.

</details>

<details>
<summary>Can Exif Editor add GPS coordinates to photos that don't have location data?</summary>

Yes. GPS coordinates add to images that were captured without location data.

</details>

<details>
<summary>Does Exif Editor work with HEIC files?</summary>

Yes. HEIC image format from iPhone and modern cameras is supported for metadata viewing and editing.

</details>
